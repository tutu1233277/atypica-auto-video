# Atypica 自动化生视频项目续接说明

## 项目目标

做一个基于现有素材自动生成短视频的工具，主要服务于 Atypica 的 TikTok / Instagram UGC 内容生产。

当前方向：

- 用 `atypica自动化生视频` 目录下已有素材自动排列组合
- 自动加中英双语字幕
- 参考 TikTok / IG 爆款 AI 工具 UGC，为 Atypica 自动生成脚本
- 视频生成底层使用 `Remotion`，后续可继续手改

## 已确认的约束

- 只基于现有素材出片，不新增高成本拍摄方案
- 不做街头采访、街拍、追路人这类高执行成本内容
- 内容方向优先低成本 UGC：
  表情 Hook + 拍电脑屏 + 报告结果 + 品牌收尾
- 参考内容主要面向北美 TikTok / Instagram Reels

## 当前已完成

### 0. 2026-05-07 最新续接重点

本轮对话的核心不是单条视频，而是把自动生视频项目产品化给团队使用。

最新方向：

- 做成内部产品，而不是只做 Codex skill。
- Skill 用来沉淀脚本审查标准。
- Web 工具用来给团队成员实际生成、审查、预览、渲染视频。
- 当前已经做出本地前端原型。

本轮用户明确过的判断标准：

- 字幕文案不要直接提 `Atypica`，会太像营销。
- 视频本质仍然要宣传 Atypica 的能力，但要像 creator 分享发现。
- Hook 要符合 TikTok 风格，优先秘密感、惊讶感、老板/团队 tension、差点犯错。
- 推荐结构是约 `12s`：`0-3s` 真人惊讶/秘密感 hook，后面直接接产品流程画面。
- 不要让脚本听起来像“我自己去 TikTok/IG 看评论就好了”。
- 产品价值要体现为：规模化抓取信号、结构化总结、深度追问、报告/决策输出。
- `AI Research` 和 `AI Interviews` 是两个不同功能，不能混在一起审。
- `AI Research` 适合市场验证、市场机会、竞品分析、用户需求挖掘、功能验证、商业决策支持。
- `AI Interviews` 适合人设选择和深度追问，也可以用于面试求职、两性关系这类高流量个人 case。
- 竞品分析只是一个 case，不要所有脚本都围绕竞品分析。

- 自动视频生成已经能跑

1. 用网页搜索自动找 TikTok / Instagram 爆款 AI UGC 链接
2. 自动追加到 [tiktok ig视频链接.md](/Users/daerduotutu/Documents/Obsidian%20Vault/tiktok%20ig视频链接.md:1)

整条链路打通。
